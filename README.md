# 杭州智慧旅游 WebGIS 交互地图

基于 OpenLayers 开发的杭州旅游景点交互地图，支持景点浏览、绘图标注、空间查询等功能。

## 功能列表

- 🗺 **底图切换**：支持高德地图 / OSM 底图切换
- 📍 **景点展示**：加载杭州 3A/4A/5A 级景点，按等级用不同颜色图钉显示
- 🔍 **景点筛选**：按类别（风景区/寺庙/湖泊/公园/山/桥）筛选景点
- 🔎 **景点搜索**：按名称搜索景点并定位
- ✏️ **绘图工具**：支持绘制点、线、面、圆，可编辑、选中、删除
- 💾 **数据持久化**：绘图数据自动保存到 localStorage，刷新后不丢失
- 📤 **导出 GeoJSON**：将绘制的图形导出为标准 GeoJSON 文件
- 📥 **导入 GeoJSON**：支持导入外部 GeoJSON 文件加载到地图
- 📏 **测距/测面积**：选中线段或多边形，计算真实距离/面积
- 🔵 **空间查询**：绘制范围图形，查询范围内的景点列表

## 技术栈

- **OpenLayers 10**：地图渲染、要素交互、坐标转换
- **原生 JavaScript（ES6+ 模块化）**：无框架，按功能拆分模块
- **Vite**：开发构建工具

## 项目结构

```
├── index.html
├── main.js          # 入口文件
├── css/
│   └── style.css
├── data/
│   └── spots.json   # 景点数据
└── js/
    ├── map.js       # 地图初始化、图层管理
    ├── draw.js      # 绘图、编辑、选中、删除
    ├── spots.js     # 景点加载、筛选、搜索、弹窗
    ├── storage.js   # localStorage 保存/加载、导入导出
    ├── measure.js   # 测距/测面积
    └── query.js     # 空间查询
```

## 本地运行

```bash
npm install
npm run dev
```

## 已知限制

- 圆形图形保存到 localStorage 或导出 GeoJSON 后，重新加载会转为多边形近似（GeoJSON 标准不支持 Circle 类型）
- 景点密集区域文字标注存在重叠，后续计划引入聚合图层（Cluster）优化
